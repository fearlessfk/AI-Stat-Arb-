基于多 Agent 协作的量化策略猎头与回测评估系统。聚焦于**加密货币长尾市场**（Top 50-500 中小市值代币）的**配对交易**（Pairs Trading），通过 AI 自动化完成从"前沿学术论文检索"到"策略代码生成"，再到"样本外盲测验证"的全流程闭环。

---

## 项目解决的核心痛点

传统量化投研面临极其严重的**人力与算力错配**问题，本项目利用 AI 解决了以下三大痛点：

### 策略情报处理瓶颈（信息过载）
全网（arXiv、SSRN、GitHub）每天涌现大量量化模型，依靠人力根本无法实时阅读、甄别并提取其中复杂的数学逻辑（如卡尔曼滤波、Copula 联结函数）。

### 长尾市场的数据维度爆炸
在长尾代币池中寻找协整对（Cointegration），每周需对近 **16,000 个代币对**进行滑动窗口检验。传统手段难以在海量数据中高效筛选并解释背后的驱动逻辑（Narrative Driver）。

### 过拟合与实盘塌方陷阱
散户和传统脚本跑出的策略往往存在严重的生存者偏差和未来函数陷阱，导致样本内（In-sample）夏普比率极高，而样本外（Out-of-sample）实盘秒亏。亟需一套脱离主观情绪的自动化交叉验证系统。

## 核心架构：四阶段多 Agent 协作工作流

本项目摒弃了传统的单体脚本，构建了基于 LLM 的 **4 阶段多 Agent 协作工作流**（Multi-Agent Pipeline），重度依赖大模型的长链推理能力：

### Agent 1：策略情报员（Discovery Scraper）

- **行为**：定时扫描学术数据库和开源社区，抓取最新的统计套利研究
- **推理**：利用 LLM 对长文本论文进行信息抽取，提炼出核心数学公式、适用特征（如 Orderbook 深度、资金费率）以及前置假设条件

### Agent 2：可行性分析师（Feasibility Grader）

- **行为**：对 Agent 1 提取的策略进行"逻辑盲审"（不跑代码，纯依靠逻辑推演）
- **推理**：结合系统现有的执行环境（例如 Maker 返佣机制、API 延迟），LLM 执行多维打分（数学严谨性、执行门槛、数据可获得性）。若判断某策略极其依赖万分之一以下的低手续费而当前系统不支持，则直接打回，避免浪费极高成本的底层回测算力

### Agent 3：代码转化架构师（Code Translator）

- **行为**：将通过 80 分阈值的策略"研报"，转化为 Python 代码
- **推理**：严格遵循内部的 BaseStrategy 接口约束，将自然语言逻辑转化为基于 Pandas/NumPy 的向量化运算代码，并自动剥离策略逻辑与物理会计账本（手续费、滑点结算），确保接入底层引擎时的绝对解耦

### Agent 4：归因与复盘专家（Post-mortem Analyst）

- **行为**：在底层引擎完成 Walk-forward（向前行走）滚动盲测后，接管长达万行的回测日志
- **推理**：执行事后归因分析。例如，通过长上下文理解找出策略失效的根本原因（"本次回撤不是因为模型失效，而是因为长尾簇某代币发生 Rug Pull 导致的联动风险"），并自动更新策略库的淘汰状态

## AI 驱动的具体成果与价值杠杆

| 指标 | 传统方式 | 本系统 |
|------|----------|--------|
| 策略从构思到验证周期 | 1-2 周 | **2 小时** |
| 认知沉淀 | 个人经验，难以复用 | **失败归因报告 + Alpha 知识库** |
| 交叉验证 | 主观选择，易过拟合 | **自动化 Walk-forward 盲测** |

### 投研效率指数级跃升
将一个新策略从"论文构思"到"代码落地并完成 7 折盲测验证"的周期，从传统人力的 1-2 周压缩至 **2 小时**。

### 高质量的认知沉淀
系统不仅仅输出代码，所有失败的回测记录均会被 Agent 4 总结为"失败归因报告"并沉淀在本地工作区，形成团队专属的 Alpha 知识库。

### API / Token 需求
由于系统中包含大量的学术论文解析（长文本）、代码多轮迭代修改（严格的格式约束）以及基于海量日志的归因推理，本项目对大模型的**长上下文窗口**（Long Context）、**复杂指令遵循**（Instruction Following）和**并发推理能力**有着极高的且高频的需求。

## 快速开始

### 环境要求

- Node.js >= 18

### 安装与启动

```bash
# 克隆项目
git clone https://github.com/fearlessfk/AI-Stat-Arb-.git
cd AI-Stat-Arb-

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入你的 API 配置

# 启动服务
node server.js
```

然后访问 **http://localhost:3000**

### 环境变量配置

```env
# DeepSeek AI
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_API_KEY=你的密钥

# 搜索提供者（可选）
SEARCH_PROVIDER=tavily        # 或 searxng / mock
TAVILY_API_KEY=你的密钥
SEARXNG_BASE_URL=你的实例地址
```

> 不配置搜索和 AI 时，系统自动降级为本地样例模式。

## 使用流程

1. **输入策略** — 填写策略名称与自然语言描述，选择市场、频率、风格
2. **智能拆解** — 系统自动将描述分解为结构化字段
3. **检索候选** — 联网搜索全网相似策略，聚合多源结果
4. **AI 评分** — DeepSeek 基于五大维度给出综合评分
5. **决策建议** — 输出"是否值得进入真实回测"的研究建议
6. **历史追溯** — 所有分析记录自动存档，支持后续复核
